Windows slaves can be launched using SSH or JNLP. Linux slaves can be launched using SSH. For Windows slaves, if you want to use SSH then the image needs to be custom-prepared with an SSH server preinstalled.

When using JNLP launch, ensure following
1) Hudson URL (Manage Hudson --> configure system --> E-mail Notification --> Hudson URL)
    This URL needs to be reachable by slave, so make sure to configure the firewall rules accordingly.
2) TCP port for JNLP slave agents (Manage Hudson --> configure global security --> Enable security --> TCP port for JNLP slaves).

    This port needs to be reachable from slave launched using JNLP. It is recommended to use a fixed port so that necessary firewall exceptions can be specified.

    If Hudson master is running on Azure, then open an endpoint for "TCP port for JNLP slave agents" and add the necessary firewall rules (Run --> firewall.cpl) inside the virtual machine.
